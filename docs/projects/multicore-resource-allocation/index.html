Multicore Resource Allocation

Designing and evaluating contention-aware schedulers and isolation strategies so multicore systems deliver predictable performance under mixed workloads.

Overview

Why this matters

Multicore platforms routinely multiplex latency-critical, throughput-oriented, and best-effort workloads. Without careful allocation and scheduling, cache and memory contention can break performance contracts and destabilize systems.

This project aims to characterize these contention patterns and develop mechanisms—at both OS and runtime layers—to deliver predictable execution while keeping systems efficient.

Initial objectives

  • Survey hardware contention points (LLC, memory controllers, interconnect).
  • Prototype scheduling policies that adapt to observed interference.
  • Build repeatable benchmarks to measure isolation vs. utilization tradeoffs.
  • Document trace formats, setup scripts, and evaluation methodology.
